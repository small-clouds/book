多线程

进程/线程

进程：系统运行程序的基本单位 有独立的内存空间和系统资源
线程：进程中执行运算的最小单位 处理机分配给线程 即真正在处理机上运行的是线程
创建线程对象 线程 分为主线程 和子线程 并行关系 最先启动的 为主线程 主线程 关闭 子线程 也可能成为主线程 并无明确的界限

创建线程的方式

继承Thread 类 重写run方法
实现Runnable 接口 实现run方法
创建线程对象 打点调用start();

线程的状态与运行

线程的四种状态 新生状态 -> start -> 可运行状态 -> ( 阻塞状态 ) run -> 死亡状态
阻塞原因 1.io流阻塞 2.人为睡眠 3.线程锁

线程调度

join： 等待该线程终止的时间 ms a线程 调用b线程的join方法 a暂停 等待b执行啊无奈
Yeild: 暂停 级别挺低的 暂停当前线程 执行其他线程
Sleep: 挂起 有参数毫秒值 暂停 专业名词 挂起

加锁(Synchronized)

可以放在 同步代码块上 放在这最好 串行化 范围越小越好
