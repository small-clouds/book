新建简单的web流程

new->web project
webRoot/WEB-INF/web.xml Servlet的核心配置文件
Url-pattern 访问路径
Servlet-name 是标记 只要与 上面的 servlet 中的 servlet-name 一致即可只是用来找到处理类的 servlet-class 必须是全名称
在src下面新建servlet 会在核心配置文件生成配置
新建一个servlet mapping url 后缀加.do 目的是为了区分 POJO(project old java Object) mapping url是访问该servlet 的地址

部署项目的n种方式

复制的项目一定要更改文件的路径 否则报错
向服务器webapps中放入web工程
部署时，将源代码编译成字节码文件放到 web-inf/classes下面
以上有两种方式通过工具/手动向webapps放入项目
将war包放入到webapps下/启动tomcat 启动的过程就是解压war包的过程
jar并不算是一种部署项目的方式 但是能够实现

修改项目的访问路径

复制过来的项目一定要修改访问路径
项目右键 properties 搜索web 第一个web更改路径

jar包和war包

jar包：选择src目录 -> exprot ->java -> jar file
war包：选择所有文件 ->exprot ->javaee ->war file
