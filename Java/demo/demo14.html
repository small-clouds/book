内部类 垃圾回收 异常

垃圾回收

垃圾： 没有指针 引用的对象 叫做垃圾 万物皆对象
垃圾回收线程(守护线程):遇到作用域结尾时 通知清理 {} 作用域
finalize 回收 System.gc(); 通知 会去执行finalize

内部类 --反正我没搞懂分为四种目的 隐藏类文件

静态内部类 static 不能修饰根类

    public class A{
        static private int num =1;
        static class B{
        }
    }

    调用:   A.B a = new A.B();

成员内部类

    public class A{
            private int num =1;
            public	class B{
                System.out.print(num);
            }
    }
    调用:  
        A a = new A(); 
        A.B b = a.new  B();

方法内部类

注：方法内的 内部类访问时 必须final 修饰 
    public class A{
        private int num =1; 
        public void aMethod(){
            final int num2 = 2; //方法内的 内部类访问时  必须final 修饰 
            public	class B{
                System.out.print(num);
                public void bMethod(){
                        System.out.print(num);
                }
            }
        调用 必须 在这创建
        B  b = new B();
        b.bMethod()
        }
    }
方法一结束 类就消失了

匿名内部类

    public  class  A{
        public void run(){
            System.out.print("run .........");
        }
    }
    调用 :  A  a = new A(){
        public void run(){
            System.out.print("run .........");
        }
    };
注：a.run(); 就相当于重写了方法
