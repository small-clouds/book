MVC介绍

MVC

(M(model) V(view) C(controller))是架构层面的
gof23种设计模式是代码层面的
有助于管理复杂的应用程序 可以在不依赖视图的情况下去处理业务逻辑 也可以不依赖业务逻辑去处理视图

项目设计流程

控制层-->获取用户数据--servlet
业务层-->应用程序的业务逻辑--Service
持久层(数据访问层 DAO)对数据库操作--对数据库表的增删改查
实体类(model 表 类 ) bean entity domain pojo
工具包 (整个通用工具类)--bean entity domain pojo
注意：控制层分层不明确
控制层 页面html/jsp servlet 都叫控制层
控制层 页面html/jsp 就是视图 servlet 都叫控制层

M model模型

是应用程序中处理应用程序数据逻辑的部分 通常负责在数据库中存取数据
Model1(过时) 在页面上可以编写java代码
Model2 全标签

V view视图

应用程序中处理数据显示的部分 通常视图是一句模型数据创建的
前端 html/jsp

C controller 控制器

是应用程序中处理用户交互的部分 负责从视图读取数据控制用户输入 向模型发送数据
获取用户数据 servlet
Struts2 Struts1 springMVC 都是以servlet为基础的
